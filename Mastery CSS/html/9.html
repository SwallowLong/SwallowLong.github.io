    本章主要学习了以下内容：

    1. 如何跟踪CSSbug
    2. 神秘的hasLayout
    3. hack和过滤器
    4. 最常见的bug和修复方法
    5. 分级浏览器支持

    自己多注意不要使用错误的css和html，也不要忽略了外边距的一些问题，真要说什么大体，我TM也说不出来，毕竟我并没有太多修复bug的体验，只能待以后再来补充了，还可以在一些活跃的css社区寻求帮助，当然不是随便问，你要通过自己的测试案例缩小范围，找到大概位置，不然几百行代码谁愿理你？最好加上截图，加上个准确的标题，而且准确的标题有时候还能帮你在搜索引擎中搜索问题，毕竟可能这个问题不是你单独有，你说呢？

    不要使用太过复杂的html和css，这很容易出问题，而且前面也说过，如果过于复杂的话，可能是提醒你，你的html结构可能有点问题

    一些有名点的例子：IE的双外边距浮动bug，加个display: inline，还有就是注意IE中的布局问题，以及3像素文本偏移bug，通过设置一个小的height(这是在IE6及更低版本，不影响，其他浏览器注意隐藏)使其布局，然后将浮动文本margin-right: -3px;(如果浮动的是图像的话，还要将图像margin: 0 -3px;),具体怎么注意，还待以后摸索

    hack：就是不同浏览器有不同的识别规则，这个问题是不可避免的，也可以通过这点达到在不同的浏览器中能显示出不同的页面效果

    分级浏览器支持：通过测试和hack达到不同的浏览器不同效果，当然不是要所有的都表现很好，可能有的表现差点也是允许，至于那些表现差，差多少可以接受，就看你自己心里的分级标准
