    • 第三章

    本章主要学习了以下内容：

    1. 盒模型的复杂性和特点
    2. 如何以及为什么使用外边距叠加
    3. 绝对定位和相对定位的差异
    4. 浮动和清理是如何工作的

    盒模型：由外到内分别是外边距margin、边框border、内边距padding和最后也是最重要的内容区域，一般来说width指的就是内容区域的宽度，为什么是一般来说呢，因为在早期的IE，包括IE6中width指的都是内容区域+padding

    外边距：外边距主要要注意叠加的问题，这个都是老生常谈了，我也不多讲，这里着重提下包含框中的margin问题，因为TMD这个也是会合并的，你因为在垂直方向上应该是内容的margin+div的margin，实则并不是，高度就是内容的高，因为垂直方向会合并，水平不会，如果你想达到你想要的高度，那也很简单，设置border(这里当然指的是div的，不要理解错误)或是给div设置padding那就OK，合并的问题只是只对块级，对于行内、浮动、绝对定位是没什么影响的

    定位：绝对定位就是相当他最近一个定位的父元素，可以是relative也可以是absolute，而相对定位relative相对的是他原来文档流中的初始位置，一般都是父元素relative和子元素absolute配合使用

    浮动和清理：浮动是一个很不错的东西，当然只是我个人观点，他在文档流就和absolute一样是不存在了的，而clear也不是怎样，只是给他留了个垂直外边距，不得不说float的外边距是很重要的一个特性，而由于float是不占文档流中的位置，所以他的父框自然是0了，也就是不会显现，你可以在float文本后加个空的div让其clear，或者使用overflow属性，我个人是比较喜欢后者的
